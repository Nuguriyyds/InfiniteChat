**InfiniteChat 分布式即时通信系统** | 2025.12-2026.02 | 独立开发 / Java 后端

**背景**：面向即时通信与轻社交场景，围绕高频发送、低延迟推送、消息可靠性与在线状态一致性等核心问题，设计可验证、可压测、可恢复的分布式 IM 后端架构。

**目标**：构建基于 Spring Cloud + Netty + RocketMQ + Redis 的分布式消息链路，实现会话级有序投递、在线精准路由、离线补偿及高并发红包处理，并通过压测量化验证系统瓶颈与故障恢复能力。

**过程**：
- 构建 Netty WebSocket 接入层，采用 Boss / Worker / Business 三层线程模型，实现单设备在线与跨节点精准推送；节点重启后通过指数退避 + 随机抖动重连，将恢复阶段失败重试从 2000 次降至 126 次。

- 实现消息主链路幂等与会话级有序语义，使用 Redis Lua 原子去重与会话级序号分配，结合 RocketMQ 顺序分片投递、异步落库与 failover 重试，保证“至少一次投递 + 消费端去重”；主导 3 轮 wrk2 压测与调优，定位事务长占连接、伪同步 ACK、Redis 连接池不足等瓶颈，将发送入口从 491 RPS / P99 48s 优化至 994 RPS / P99 80ms。

- 设计离线消息热冷分层存储，推送失败消息回流 OfflineService，按 Redis ZSet（3 天热数据）+ MySQL（7 天冷数据）双写，并支持基于 `sessionId + seq` 的增量补拉与未读计数维护。

- 实现红包事务链路，发红包通过 MySQL 行锁 / CAS 完成余额扣减，抢红包使用 Lua 原子脚本 + RocketMQ 事务消息保证 Redis 预扣与 MySQL 落库最终一致，防止超发和重复领取；红包接口在 800 RPS 下 P99 51ms、零错误，一致性验证零超发、零重复领取。

**结果**：在 2C8G 单节点环境下，RTC 1000 长连接稳定在线，高频心跳场景无异常断连，节点重启后约 8s 恢复至 100% 活跃连接；消息发送入口在 1000 RPS 目标压测下实测 994 RPS、P99 80ms、零超时零错误；红包接口在 800 RPS 下 P99 51ms、零错误，一致性验证零超发、零重复领取。

**技术栈**：Java、Spring Boot、Spring Cloud Gateway、Netty、RocketMQ、Redis / Redisson、MySQL、MyBatis-Plus、Nacos、OpenFeign、JWT
